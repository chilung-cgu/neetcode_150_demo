# 貢獻指南 (Contributing Guide)

## 📝 Markdown 寫作規範

為了確保文檔在 MkDocs 網站上正確渲染，請嚴格遵守以下格式規則：

### 1. 列表 (Lists)
* **規則**：在任何列表（有序 `1.` 或無序 `-`）之前，**必須保留一個空行**。
* **原因**：MkDocs 使用的解析器若無空行，不會將其識別為列表。

**❌ 錯誤範例**：
```markdown
解釋如下：
1. 第一點
2. 第二點
```

**✅ 正確範例**：
```markdown
解釋如下：

1. 第一點
2. 第二點
```

### 2. 數學公式
* 使用 `$E = mc^2$` 語法。
* 確保不與前後文字黏連，建議前後留空格。

### 3. 圖片與資源
* 圖片請放在 `assets/` 目錄。
* 使用絕對路徑引用或相對路徑。

## 🔍 本地測試
提交代碼前，請務必在本地運行：
```bash
mkdocs serve
```
並檢查 http://127.0.0.1:8000 的顯示效果。
